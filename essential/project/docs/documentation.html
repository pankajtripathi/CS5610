Goal:

The objective of this web development project is to design a website where user can check out the movies currently in theaters, upcoming movies, movies currently topping box office charts, search movies and along with these features user can login and create his own list of movies to watch.

System Features:

1. Search Movie

This feature allows user to search for hisher favorite movie from the RottenTomatoes API and check the movie details.

2. In Theaters

This feature provides user with all the movies currently showing in theaters that he/she can check out. User can click on the result and check the movie details

3. Box Office

This feature provides user with movies currently topping box office charts. User can click on the result and check the movie details

4. Opening Movies

This feature allows user to check out movies opening this week. User can click on the result and check the movie details

5. Upcoming movies

This feature allows user to check out movies coming next week and month. User can click on the result and check the movie details

6. Movie Reviews

This feature allows user to check out movie reviews on the movie details page.

7. Login

This feature enables user to login and create his/her own list of movies to watch

Login Page

Login Page:

The login page is provided in order to enable user to login into his account and add movies to his watch list. Once user logs in he/she can search and add movies he/she wishes to watch sometime later and once done watching the list can be updated by deleting the movie.




Login Controller

Login services are designed using Node.js and Express Module. MongoDB is used for storing all the user data. Since the web page is implemented as a single page website, angular routing is used to route pages and for login a controller is used to complete login action.

Registration Page

Registration Page

I have implemented registration page to enable new user to register himself/herslef in order to access and create a new profile so that he/she can add movies to his/her watch list. I have created a basic registration form where user enters generic details and get registered.






Registration Controller

I am registering a new user are created using NodeJS, AngularJS and MongoDB. The following sevices are playing a role on this page. Check if user name exists or not. Register a new user, store all the details in DB and respond back accordingly

Data Storage: MongoDB

Data Storage: MongoDB

A lot of data is handled in the project, for backend storage purpose I am using MongoDB which is a no SQL database. In mongoDB there are collections instead of tables and documents (JSON Objects) instead of tupples. The Coolections for my project are :

  • Usermodels: This collection is used to store the user details.
  • Movmodels: This collection is used to store the users movies in movie watchlist along with the username.
